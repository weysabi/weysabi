---
title: Prompt management
description: Define reusable prompt templates and execute them through the Weysabi pipeline.
---

Weysabi has two prompt paths:

- The SDK prompt registry is for code-defined prompts shipped with an application.
- The server control plane is for persistent, project-scoped prompt drafts, immutable versions, and
  publishing.

## Register a prompt

```ts
import { z } from "zod";

sabi.prompts.register({
  id: "classify-ticket",
  messages: [
    {
      role: "system",
      content: "Classify customer support tickets.",
    },
    {
      role: "user",
      content: "Ticket:\n\n{ticket}",
    },
  ],
  model: "groq/llama-4-scout",
  temperature: 0.2,
  maxTokens: 80,
  schema: z.object({
    category: z.enum(["billing", "technical", "account", "feature"]),
  }),
});
```

## Render and execute

```ts
const result = await sabi.prompts.run("classify-ticket", {
  ticket: "I was charged twice this month.",
});

console.log(result.parsed?.category);
```

## Runtime overrides

```ts
const result = await sabi.prompts.run(
  "classify-ticket",
  { ticket },
  {
    model: "openai/gpt-4o-mini",
    temperature: 0,
  }
);
```

## Registry operations

```ts
sabi.prompts.registerMany(definitions);
sabi.prompts.has("classify-ticket");
sabi.prompts.get("classify-ticket");
sabi.prompts.list();
sabi.prompts.remove("classify-ticket");
```

<Callout type="info" title="Registry versus control plane">
  The registry is in-memory and application-owned. Persistent drafts, immutable versions,
  publishing, rollback, and project isolation belong to the server control plane.
</Callout>

## Managed prompt versions

Use the HTTP client from backend code when prompts should be edited, published, or rolled back at
runtime.

```ts
import { createSabiClient } from "@weysabi/sabi/client";

const sabi = createSabiClient({
  baseUrl: process.env.SABI_BASE_URL!,
  apiKey: process.env.SABI_ADMIN_API_KEY!,
});

const project = sabi.project(process.env.SABI_PROJECT_ID!);

const prompt = await project.prompts.create({
  name: "Support Agent",
  slug: "support-agent",
});

const version = await project.prompts.versions.create(prompt.id, {
  model: "openai/gpt-4o-mini",
  messages: [
    { role: "system", content: "You support {company}." },
    { role: "user", content: "{message}" },
  ],
});

await project.prompts.versions.publish(prompt.id, version.id);
```

Execute the published prompt through the managed conversation endpoint:

```ts
const result = await project.conversations.messages.send(conversationId, {
  content: "My invoice is wrong",
  prompt: "support-agent",
  promptInputs: { company: "Acme" },
});

console.log(result.assistantMessage.content);
```
